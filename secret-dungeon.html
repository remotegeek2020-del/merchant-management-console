Prime49 Task Automation
Auto-create tasks in Task Center when new Prime49 merchants are uploaded via CSV
Super Admin Only

When ON, a task is created for every brand-new Prime49 merchant on each upload
Can be changed on any task afterwards
Placeholders: {{dba_name}} {{mid}} {{agent_id}} {{partner_name}} {{enrollment_date}} {{account_status}}
Same placeholders apply. Line breaks are preserved.
How It Works
  • Triggered every time merchants are uploaded via the CSV upload on the Merchants Dashboard
  • Only brand-new merchants are checked — existing records being updated are skipped
  • New merchants whose Agent ID is flagged as Prime49 in the system trigger this automation
  • One task is created per new Prime49 merchant using your configured title, description, assignee, and priority
  • Tasks appear instantly in the Task Center linked directly to the merchant record

ShipStation-Ready Deployments — New Ticket Mode
Controls what the deployment dashboard's + New Ticket button shows. Pick one mode:
• Disabled — opens the current standard deployment form directly (no ShipStation, full fallback).
• Coming Soon — shows the wizard with ShipStation Ready greyed out + a "Coming Soon" badge; standard Create Ticket still works.
• ShipStation Only — wizard goes straight to ShipStation Ready; the plain Create Ticket option is hidden.
• Both — wizard shows both cards active; staff choose per ticket.

ShipStation-Ready Returns — Mode
Controls the return flow (from the deployment edit modal's Log Return). Pick one mode:
• Disabled — standard return flow only (full fallback).
• Coming Soon — return method prompt shows ShipStation greyed; standard return works.
• ShipStation Only — every return goes through the ShipStation return modal.
• Both — staff pick Standard or ShipStation return each time.
