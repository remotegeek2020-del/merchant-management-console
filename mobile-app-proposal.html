Executive Summary

PayProtec already operates a fully functional web-based Merchant Management Console — a powerful internal tool used daily by staff to manage merchants, deployments, RMAs, support tickets, and equipment inventory. This proposal recommends extending that platform into a native mobile application, giving staff the ability to perform their most critical work directly from their phones — no laptop required.

The business case is straightforward: 80% of today's global workforce is deskless (IDC, 2024), and organizations that equip their teams with mobile tools report an average 27% increase in staff productivity. For PayProtec, this means faster RMA filing in the field, real-time merchant status checks, and fewer delays caused by staff being away from a desk.

Implementation Plan

We recommend building the app using React Native + Expo — a proven cross-platform framework that produces a single codebase deployable to both iOS and Android simultaneously. This is the same approach used by Walmart (96% code reuse), Meta, and Shopify. 78% of Fortune 500 companies now use cross-platform frameworks for their internal tools.

Backend (No Changes Required)
  • All existing Vercel API endpoints reused as-is
  • Supabase database and authentication unchanged
  • GHL, Postmark, and ShipStation integrations carry over automatically
Mobile App — Core Screens
  • Secure login + session management (existing staff accounts)
  • Merchant search, profiles, and status overview
  • Returns / RMA dashboard — file, view, and complete RMAs on the go
  • Deployments dashboard — view and update deployment status
  • Activity logs — read-only audit trail
Mobile-Specific Enhancements (New Capabilities)
  • Camera barcode scanner — scan serial numbers to instantly look up or file an RMA
  • Push notifications — alert staff when RMA status changes or deployment is due
  • Offline read cache — view merchant and equipment data without a signal
  • Role-based access — same permissions as the web console

Testing Strategy

A structured testing approach ensures the app performs reliably for staff in real-world conditions — not just in the office on WiFi.

Functional Testing
  • Test all API calls over cellular (4G/5G), not just WiFi
  • End-to-end RMA filing flow from a mobile device
  • Deployment creation and status update on mobile
  • Barcode scan → serial number lookup accuracy
Device & Compatibility Testing
  • Test on real physical devices — not just simulators
  • Small screen (iPhone SE) and large screen (Samsung Galaxy S/iPad)
  • iOS 16+ and Android 11+ minimum
Security Testing
  • Session tokens stored in encrypted SecureStore (not plain AsyncStorage)
  • Session expiry + forced re-login behavior verified
  • Role-based access enforced — staff cannot see admin screens
  • All API traffic over HTTPS; no sensitive data cached unencrypted
Beta Testing
  • Internal beta with 2–3 staff members for 1–2 weeks before launch
  • Collect feedback, identify gaps, fix before wider rollout
  • Crash reporting enabled from day one (Sentry or Expo's built-in)

iOS vs. Android — Platform Recommendation

We recommend building for both platforms simultaneously using a single React Native codebase. However, for the initial rollout, Android is the faster path to distribution — especially if we go the internal-only route (which avoids app store reviews entirely).

Factor iOS (App Store) Android (Google Play)
Review Time 1–3 days avg. (up to 7) Hours to 1 day
Rejection Rate ~25% (Apple, 2024) ~10–15%
Developer Account $99 / year $25 one-time
Review Process Mandatory human review Mostly automated
Internal Distribution TestFlight (no full review) Direct APK install (no store needed)
Ease for Internal Apps Moderate Easiest
Recommended approach — Skip the app stores entirely for internal staff: Distribute via TestFlight on iOS and a direct APK download link on Android. Staff install it like any other app. No Apple review, no Google review, no wait time. This is how most enterprise internal tools are distributed.
